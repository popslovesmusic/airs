# .gitignore Configuration Summary

## Purpose
This .gitignore is configured to exclude rebuilable artifacts while preserving essential dependencies and test results.

## Files EXCLUDED (rebuilable)

### Build Artifacts
- `build/` - All CMake build output (can be rebuilt)
- `*.obj`, `*.o` - Object files
- `*.exp` - Export files
- `*.exe` - All executables (rebuilable from source)
  - **Exception**: Backup executables kept (`.backup_*`)

### Engine DLLs (rebuilable)
- `Simulation/bin/dase_cli.exe` - Rebuilable
- `Simulation/bin/dase_engine*.dll` - Rebuilable
- `Simulation/bin/*step_runner.exe` - Rebuilable

### Temporary/Cache Files
- `cache/`, `*.cache`
- `__pycache__/`, `*.pyc`, `*.pyo`
- `logs/` - General log files
- `workspace/temp/`, `workspace/output/`
- `*.tmp`, `*.temp`

### Downloaded Packages
- `fftw-3.3.5-dll64.zip` - Can be re-downloaded
- `bench*.exe`, `fftw*-wisdom.exe` - FFTW utilities

## Files PRESERVED (essential)

### Third-Party Libraries (CRITICAL - cannot rebuild)
✅ `Simulation/libfftw3-3.lib` - Import library for FFTW
✅ `Simulation/libfftw3-3.dll` - FFTW runtime library
✅ `Simulation/bin/libfftw3-3.dll` - Deployed FFTW DLL
✅ `*.def` - All definition files (needed for lib generation)

### Test Results & Validation
✅ `harness_full_results.log` - Complete test harness results
✅ `harness_test_results*.log` - All test result logs
✅ `artifacts/validation/**/*.json` - All golden reference files
✅ `metrics/**/*.json` - All metrics output

### Test Fixtures
✅ `test_*.json` - Test output files
✅ `fixtures/**/*.json` - Test input fixtures
✅ `validation*.db` - Validation databases

### FFTW Build Dependencies
✅ `libfftw3-3.def` - FFTW definition file
✅ `libfftw3f-3.def` - Single-precision FFTW
✅ `libfftw3l-3.def` - Long-double FFTW
✅ `fftw3.h` - FFTW header file

### Configuration & Documentation
✅ All `.md` files
✅ All `.txt` files in root
✅ `Simulation/tests/fixtures/` - All test fixtures
✅ `config/` - Configuration files (except local.json)

## Critical Files That MUST Be Committed

After building, these files should be added to git:

```bash
git add Simulation/libfftw3-3.lib
git add Simulation/libfftw3-3.dll
git add Simulation/bin/libfftw3-3.dll
git add libfftw3-3.def
git add harness_full_results.log
git add artifacts/validation/**/*.json
git add metrics/**/*.json
```

## Rebuilding After Fresh Clone

1. **FFTW library files will be present** (committed)
2. **Run CMake configuration**:
   ```bash
   cmake -S Simulation -B build -G "Visual Studio 17 2022" -A x64 -DBUILD_HARNESS=ON -DBUILD_TESTS=ON
   ```
3. **Build all targets**:
   ```bash
   cmake --build build --config Release
   ```
4. **Copy step runners to Debug for tests**:
   ```bash
   mkdir -p build/Debug
   cp build/Release/*step_runner.exe build/Debug/
   cp build/Release/*.dll build/Debug/
   cp Simulation/libfftw3-3.dll build/Debug/
   ```
5. **Run test harness**:
   ```bash
   cd build && PATH="$(pwd)/Debug:$PATH" Release/harness_tests.exe
   ```

## Why This Configuration

### Excluded Items
- **Build artifacts**: Rebuilable from source, large, platform-specific
- **Executables**: Rebuilable, large (645 KB - 896 KB each)
- **Engine DLLs**: Rebuilable, small but version-specific

### Preserved Items
- **FFTW library**: External dependency, difficult to rebuild, critical for compilation
- **Golden files**: Reference data for validation, cannot be regenerated without known-good binaries
- **Test results**: Documentation of validation status
- **Definition files**: Needed to generate import libraries

## Notes

- Total preserved FFTW files: ~3 MB (2.6 MB DLL + 237 KB lib)
- Test results: ~100 KB
- Golden validation files: ~2 KB
- **No executables committed**: All rebuilable from source in ~2 minutes
